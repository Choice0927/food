# 美食旅行攻略 H5 应用需求规格说明书（V2.0）

## —— 基于“收藏 + 随机决策 + 轻打卡”理念改造

## 1. 项目背景与目标

### 1.1 背景

用户常常遇到“今天不知道吃什么”“周末不知道去哪玩”的选择困难。虽然手机里收藏了大量餐厅、景点，但分散在聊天记录、备忘录、点评App中，缺乏统一管理，更无法高效决策。\
同时，用户希望记录自己去过的美好体验，但“打卡”不应是首要目标，而是对收藏地点的补充留念。

### 1.2 目标

- 构建一个 **“我的私藏地点库”** ，用户可快速收藏感兴趣的美食店、游玩地，并打上自定义标签。
- 提供 **随机推荐、智能筛选** 功能，帮助用户在纠结时快速做出选择。
- 支持 **轻打卡**：对已去过的地方，可添加图文记录（可选），作为回忆或分享。
- 保留 **行程规划** 能力，将收藏地点按天串联，生成路线。
- 打造轻量级、移动端优先的 H5 应用，支持微信内打开，便于快速查阅和分享。

## 2. 用户角色

- **普通用户**：注册登录后可收藏地点、打卡、创建行程、分享地点等。
- **访客**（未登录）：仅可浏览公开的地点信息，不能收藏和规划。

## 3. 功能需求

### 3.1 用户注册与登录

| 功能点   | 详细描述                                   |
| :---- | :------------------------------------- |
| 注册    | 支持手机号 + 验证码注册，或邮箱 + 密码注册。需校验格式，密码加密存储。 |
| 登录    | 支持手机号/邮箱 + 密码登录，可选“记住我”保持登录态。          |
| 第三方登录 | 增加微信登录，支持绑定手机号。                        |
| 找回密码  | 通过注册手机号/邮箱发送验证码，重置密码。                  |
| 退出登录  | 清除本地 token，返回未登录状态。                    |

### 3.2 首页 / 探索页（改造为“我的收藏” + 推荐）

**布局**：底部 Tab 栏第一项，顶部导航栏包含**搜索框**、**地图入口**图标。\
**核心功能区**：

- 顶部大按钮 **“随缘一吃”**（或“随缘一玩”）——点击后从当前城市/标签筛选的收藏中随机推荐一个地点，弹出卡片，用户可“去这里”或“再摇一次”。
- 下方分为 **“我的收藏”**（优先展示）和 **“附近热门”**（公共地点推荐）。

| 功能点         | 详细描述                                                          |
| :---------- | :------------------------------------------------------------ |
| 搜索          | 支持按城市、地点名、标签搜索。搜索结果优先展示用户已收藏的地点，再展示公共地点。                      |
| 我的收藏卡片展示    | 展示用户已收藏的地点，按最近收藏时间倒序。每个卡片：封面图、名称、城市、标签（如“火锅”“咖啡”）、已打卡标记（小图标）。 |
| 下拉刷新 / 上拉加载 | 列表支持下拉刷新和分页加载。                                                |
| 点击卡片        | 跳转至地点详情页（见 3.3）。                                              |
| 右上角地图入口     | 点击后跳转至全局地图，仅显示用户收藏的地点（可切换显示公共地点）。                             |
| 快捷筛选        | 提供标签筛选条（如“火锅”“咖啡”“景点”），点击后只展示对应收藏。                            |
| 发布入口        | 底部悬浮“+”按钮，点击进入**新增地点**页（3.4）。                                 |

### 3.3 地点详情页（原美食详情页，核心改造）

| 功能点   | 详细描述                                                         |
| :---- | :----------------------------------------------------------- |
| 基本信息  | 展示名称、地址、人均消费、营业时间（若有）、用户描述、备注。                               |
| 图片浏览  | 支持多图滑动查看，双击放大，可保存图片。                                         |
| 位置地图  | 内嵌小型地图展示该店位置，点击可跳转至全屏地图并导航。                                  |
| 互动操作  | **收藏/取消收藏**（核心）、点赞（公共地点的热度）、分享。                              |
| 打卡记录  | 显示该地点下所有用户的打卡记录（若已打卡则显示“我的打卡”）。                              |
| 我去过   | 按钮“我去过”或“打卡”——点击后进入打卡编辑页（可写评价、上传图片、评分），打卡后该地点在我的收藏中标记为“已打卡”。 |
| 添加到行程 | 提供“添加到行程”按钮，用户可选择已有行程或新建行程。                                  |
| 相关推荐  | 同城市其他热门地点，或同标签的其他地点。                                         |

### 3.4 新增/编辑地点（原新增美食打卡，改造为添加公共地点）

| 功能点  | 详细描述                                            |
| :--- | :---------------------------------------------- |
| 名称   | 必填，文本输入框，最多 50 字。                               |
| 城市   | 必填，支持搜索城市（或自动定位当前城市）。                           |
| 地址   | 必填，详细地址，可辅助定位。                                  |
| 描述   | 选填，多行文本，最多 300 字。                               |
| 备注   | 选填，可记录推荐菜、价格、小贴士等。                              |
| 上传图片 | 支持从相册选择或拍照，最多 9 张，图片压缩上传。                       |
| 定位   | 调用 H5 定位获取经纬度，并反向解析地址；用户可手动调整位置（地图选点）。          |
| 标签   | 可选标签：美食类（火锅、日料、咖啡……）、游玩类（公园、博物馆、游乐场……），支持自定义标签。 |
| 评分   | 可选星级评分（1-5星），作为公共评价参考。                          |
| 提交   | 保存后地点进入公共库，同时自动收藏到当前用户。                         |
| 编辑   | 在详情页提供编辑入口，仅发布者本人可编辑。                           |

### 3.5 行程规划（同原稿，但添加地点来源优先从收藏中选取）

| 功能点   | 详细描述                                           |
| :---- | :--------------------------------------------- |
| 创建行程  | 填写行程名称、起止日期、目的地城市，可选封面图片。                      |

| 添加地点  | **优先从我的收藏中添加**，也可通过搜索公共地点或从收藏夹中快速添加。每个地点可设置备注。 |
| 顺序调整  | 支持拖拽调整地点顺序。                                    |
| 地图模式  | 在行程详情页切换地图视图，将所有地点标记在地图上，并显示优化后的路线（按顺序连线）。     |
| 路线规划  | 调用地图 SDK 的路线规划功能，展示从起点到各点的驾车/步行路线及预估时间。        |
| 分享行程  | 可将整个行程生成为海报或链接，分享给朋友。                          |
| 复制行程  | 允许复制他人公开的行程到自己的账户（二次修改）。                       |

## 4. UI/UX 设计规范

### 4.1 设计理念

- 整体风格：清新、温暖、有食欲感，同时保持简洁现代。
- 核心关键词：轻量、明亮、圆润、微质感。

### 4.2 色彩系统

| 用途    | 色值      | 说明                         |
| :---- | :------ | :------------------------- |
| 主色    | #FF6B3D | 橙红色，用于重要按钮、Tab 选中态、标签、强调元素 |
| 辅助色   | #FFB347 | 橙色，用于渐变背景、次要按钮、图标点缀        |
| 背景色   | #F8F8F8 | 页面背景                       |
| 卡片背景  | #FFFFFF | 白色卡片，带轻微圆角和阴影              |
| 文字主色  | #333333 | 标题、重要文字                    |
| 文字辅助色 | #999999 | 说明文字、时间、地点                 |
| 分割线   | #EEEEEE | 列表分割线、卡片边框                 |

### 4.3 组件与布局

- **圆角**：卡片圆角 16px，按钮圆角 24px，图片圆角 12px。
- **阴影**：卡片使用 `box-shadow: 0 2px 12px rgba(0,0,0,0.05)`，悬浮时加深。
- **字体**：系统默认字体，标题使用 18px 粗体，正文 14px，辅助文字 12px。
- **间距**：页面左右边距 16px，卡片内边距 16px，卡片之间间距 12px。
- **图标库**：统一使用 **Font Awesome 6** 或 **Vant 内置图标**。所有图标采用线性风格，主色调图标使用主色，辅助图标使用灰色。
- **底部 Tab**：图标+文字，选中态主色，未选中灰色。
- **加载状态**：骨架屏（Skeleton）代替传统 loading。
- **动效**：页面切换淡入淡出（300ms），按钮点击轻微缩放，列表项滑动反馈。

### 4.4 页面设计要点（新增随机决策界面）

- **首页**：顶部搜索栏 + 地图入口图标；突出“随缘一吃”大按钮（渐变色，带摇动动画）；下方为“我的收藏”列表卡片，卡片右上角显示已打卡标记（勾选图标）。
- **随机决策弹窗**：摇出地点后，显示大卡片（图片、名称、地址、标签），提供“导航”、“查看详情”、“再摇一次”按钮，底部有“标记去过”（快捷打卡）入口。
- **地点详情页**：收藏按钮明显（红心），打卡区域单独展示，显示最近打卡用户的头像和简短评价。
- **个人中心**：顶部头像卡片，统计数字横向排列，菜单列表使用图标+文字，右箭头。

### 4.5 适配与交互

- 移动端优先，基于 375px 设计稿，使用 rem 或 vw 适配。
- 所有可点击元素触摸区域不小于 44x44pt。
- 加载失败、空状态提供占位图和文案提示。

## 5. 非功能需求

### 5.1 性能

- 首页列表加载速度 < 1.5 秒。
- 地图初始化 < 3 秒，标记点渲染流畅。
- 图片上传支持批量，单张最大 10MB，自动压缩至 1MB 以内。

### 5.2 兼容性

- 支持主流移动浏览器：iOS Safari、Android 微信内置浏览器、Chrome。
- 响应式布局，适配 375px\~428px 宽度。

### 5.3 安全性

- 用户密码 bcrypt 加密。
- 接口需鉴权（JWT），敏感操作（删除、修改、发布）校验身份。
- 图片上传类型校验，防止恶意文件。

### 5.4 扩展性

- 后端接口 RESTful 设计，便于增加社交功能。
- 数据库预留字段，如地点分类、评分、人均消费等。

## 7. 技术实现建议

- **前端**：Vue 3 + Vite + Vant 组件库 + Font Awesome 6 + 高德地图 JS API 2.0。
- **状态管理**：Pinia 存储用户信息、收藏列表、行程数据。
- **后端**：Node.js + Express + MongoDB（灵活存储图片数组、JSON 字段），或 MySQL + JSON 字段。
- **鉴权**：JWT，存储于 localStorage。
- **图片存储**：阿里云 OSS 或七牛云，前端直传或后端中转。
- **地图服务**：高德地图 Web API（地理编码、逆地理编码、路线规划）。
- **部署**：前端托管至 OSS/CDN，后端部署至云服务器或云函数。

## 9. 分步开发与 TRAE 交互指南

为获得高质量、符合 UI 规范的代码，建议按以下步骤与 TRAE 交互，每次只聚焦一个功能模块，并明确视觉要求。

### 9.1 初始骨架（一次性搭建）

- 搭建 Vue 3 + Vite 项目，引入 Vant、Font Awesome、Pinia、Vue Router。
- 配置基础布局：底部 Tab、顶部导航栏、全局样式（色彩、圆角、阴影）。
- 创建 mock 数据，用于开发调试。

### 9.2 分模块开发顺序建议

1. 用户登录/注册 + 个人中心框架。
2. 地点详情页 + 收藏/取消收藏。
3. 首页（我的收藏列表） + 随机决策功能。
4. 新增地点 + 图片上传。
5. 打卡功能（我去过）。
6. 行程规划（创建行程、添加地点、地图模式）。
7. 全局地图（含收藏筛选）。
8. 搜索、标签筛选等辅助功能。
